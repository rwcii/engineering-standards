# Product Decision Records (PDRs)

PDRs document **how a product behaves** — user flows, feature behavior, UX contracts,
voice/identity. For process rules, the template, naming, and lifecycle, see
[PDR_PROCESS.md](PDR_PROCESS.md).

## How PDRs Relate to ADRs

ADRs document **how the system is built** (architecture, patterns, infrastructure). PDRs
document **how the product behaves**. The two often come in pairs: an ADR decides the
mechanism, a PDR decides the behavior that mechanism serves. Cross-reference via
`related_adrs` / `related_pdrs` in frontmatter.

## A note on universality

Product behavior is inherently project-specific, so this repository ships **no universal
PDR content** — only the process and a copy-ready [template](000-template.md). Each
adopting project starts its own PDR series at 001.
